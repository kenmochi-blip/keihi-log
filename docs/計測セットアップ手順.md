# 計測セットアップ手順（GA4 Data API ＋ Google広告コンバージョン）

ファネル分析①「クリック→登録率」を自動表示し、広告のCPAを最適化するための**一度きりの設定**。
完了後は、ダッシュボードを開くたびに自動更新され、登録のたびに自動でコンバージョン記録されます。

---

## A. GA4 Data API（/licenses に広告セッション→登録率を表示）

サーバーのサービスアカウント（SA）をGA4プロパティに「閲覧者」で共有するだけ。新しい鍵は不要（既存の `GOOGLE_SA_KEY` を流用）。

### 手順

1. **SAのメールアドレスを確認**
   - `keihi-log-proxy@keihi-log.iam.gserviceaccount.com`（CLAUDE.md記載のSA）

2. **GA4にSAを追加**
   - [GA4管理画面](https://analytics.google.com/) → 左下の歯車「管理」
   - 対象プロパティ「経費ログ」を選択 → 「プロパティのアクセス管理」
   - 右上「＋」→「ユーザーを追加」
   - メールに上記SAアドレスを入力、役割は**「閲覧者」**、「新規ユーザーにメールで通知」のチェックは**外す**（SAは受信できないため）
   - 「追加」

3. **GA4プロパティID（数値）を確認**
   - 管理 → プロパティ設定 → 「プロパティID」（例: `540386365`）
   - ※ `/licenses` のGA4リンク `a372782440p540386365` の `p` の後ろの数値と同じ
   - コードは `540386365` をデフォルトにしているので、**同じ値なら設定不要**。違う場合のみ次へ

4. **（プロパティIDが違う場合のみ）Vercel環境変数を設定**
   - Vercel → プロジェクト → Settings → Environment Variables
   - `GA4_PROPERTY_ID` = （手順3の数値）
   - 再デプロイ

### 確認
- `/licenses` を開く → 「ファネル分析」パネルに **⓪ 広告セッション → 登録** の行が表示されれば成功
- 表示されない場合：SA共有の反映に数分〜数十分かかることがある。それでも出なければ役割（閲覧者）とプロパティIDを再確認

> 補足：`trial_start` イベント数もGA4から取得して併記します（下のB-1で送信されるイベント）。

## B. Google広告コンバージョン計測（CPA最適化の本命）

「トライアル登録」をコンバージョンとして記録し、クリック→登録→CPAを広告管理画面にネイティブ表示＋自動入札に反映。

### B-1. すでに仕込み済み（コード側）
- `success.html`（トライアル確定ページ）で、トライアル登録時に自動で発火：
  - **GA4イベント** `trial_start`（A で取得・GA4で「キーイベント」に指定するとGA4側もコンバージョン化）
  - **Google広告コンバージョン**（下のB-2を設定すると送信開始。未設定なら送信しない）

### B-2. Google広告でコンバージョンアクションを作成

1. [Google広告](https://ads.google.com/) → 右上「ツールと設定」→「測定」→「コンバージョン」
2. 「＋ 新しいコンバージョンアクション」→「ウェブサイト」
3. 設定例：
   - カテゴリ：**「申し込み」**（または「見込み顧客」）
   - コンバージョン名：`トライアル登録`
   - 値：なし（または任意の見込み価値）
   - **カウント方法：「1回」**（1登録＝1コンバージョン）
4. 「保存して次へ」→「タグを自分で設定する」→「Googleタグ」を選択
5. 表示される2つの値を控える：
   - **コンバージョンID**：`AW-XXXXXXXXX`
   - **コンバージョンラベル**：`AbC-D_efGhIjKlmNoP`

### B-3. config.js に設定

`public/js/config.js` の `googleAds` を埋める：

```js
googleAds: {
  conversionId: 'AW-XXXXXXXXX',          // B-2のコンバージョンID
  trialConversionLabel: 'AbC-D_efGhIjKlmNoP',  // B-2のコンバージョンラベル
},
```

→ コミット＆デプロイすれば、以降トライアル登録のたびに自動送信。

### B-4.（推奨）GA4とGoogle広告をリンク
- GA4管理 → 「Google広告とのリンク」→ 対象アカウントをリンク
- これでGA4の `trial_start` をGoogle広告のコンバージョンとしてインポートしたり、広告の自動入札に活用可能

### 確認
- テストでトライアル登録 → Google広告「コンバージョン」画面で当該アクションのステータスが「最近のコンバージョン」になればOK（反映に数時間かかる場合あり）

## まとめ：自動化の範囲

| 項目 | 一度きりの作業 | 以降 |
|---|---|---|
| GA4 Data API（⓪行の表示） | SAをGA4に閲覧者で共有（＋必要ならプロパティID環境変数） | ダッシュボードを開くたび自動取得 |
| Google広告コンバージョン | コンバージョン作成＋config.jsに2値設定 | 登録のたび自動記録・自動入札最適化 |

どちらも設定後の運用作業はありません。

> 注意：`success.html` は広告クリック後にユーザーのブラウザで開かれる必要があります（通常のトライアル申込フローで自動的に開かれます）。LINE等のアプリ内ブラウザ経由だと計測が欠けることがあります。
